Fuzzy Matching Guide

Fuzzy matching allows you to find tabs even if you have typos or only remember part of the title. It uses the powerful Fuse.js library to calculate how "close" your search term is to each tab.

The Threshold Slider

The threshold value determines how strict the search is. A lower value is more precise, while a higher value is more forgiving.

Value Strictness Behavior
0.00 Exact Only matches the exact sequence of letters. No typos allowed.
0.10 Strict Handles minor typos (e.g., "gogle" will find "Google").
0.35 Balanced Our Default. Recommended balance between precision and flexibility.
0.60 Loose Very forgiving. Matches "jav" to "TypeScript". Often too noisy.
Pro Tip: If you find that random tabs are appearing in your results, try lowering the threshold to 0.20 or 0.25.

Why am I seeing unrelated tabs?

Sometimes common words in URLs can trigger a match. For example, at higher thresholds:

  • Searching for "twitch" might match a YouTube URL because it contains the word "watch" (both share the letters w, t, c, h) at our default 0.35 threshold, but it will be excluded if you lower the setting to 0.30.
  • Searching for "github" might also return results for "gitlab" due to their similar structure.

If this happens frequently, lowering the threshold or unchecking Search URLs will provide much stricter results.

Technical Details

Fuzzy matching is currently applied to Tab Titles and URLs. Content search (searching inside the page) remains an exact-match search to ensure performance and accuracy.
